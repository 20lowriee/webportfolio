In this unit, we learned about web graphics and how to properly add them to a web page. First, we
learned that in order to add an image found on another website, it must not violate any copyright
laws; the image must have a public domain or creative commons license. This means that the owner
of the image gave consent that their image can be used on sites other than their own. Next, we
created a web photo album using HTML/CSS. Again we used the copyright laws to find images, then
resized these images to create thumbnails to go along with the actual image. The thumbnails were
inserted into the web pages with links to the actual photo. After this was completed, we used
Photoshop in order to create buttons to add to our web pages. We practiced working with layers in
the app as well. Next, we optimized GIF images, meaning we made them the best quality in order to
add them into our web pages. Lastly, we created Favicons for our sites. At first, it was difficult
to find the correct code to make it show up when I ran my web page, but after trying many different
codes, it finally worked. Now, my star-shaped Favicon can be seen here on this webpage!

used along with HTML and CSS to enhance the web page. When we first tried it out in the tutorials on
Codecademy, it was a little bit confusing, but the more I practiced with it, the more I was able to understand

alerts, buttons to control these alerts, and a real-time digital clock using Javascript scripting. It got a little
bit frustrating at times because it was hard for me to debug since I am so new to this script. But, I finally
got my clock and buttons to work and now it is displayed on my Javascript page. Overall, this unit was fun
but challenging.

Unit VI - Quality Control
  • Module 1: Validating Websites

    • L1: Validating HTML

    • L2: Validating CSS

    • L1: Validating Accessibility

  • Module 2: Testing Usability

    • L1: Conducting a Usability Test

  • Module 3: Reflections

In this unit, we learned how to validate out website's code. Validating the code is a big part of website
production because in order for your site to work and display the same way on all browsers, the code must
comply to web standards across all platforms. With the validation software we used in class, it gave a list of
coding errors and warnings. One of the biggest things I learned from this was that even though the website looked
good when it was run on Google Chrome, it may look different on, for example, Internet Explorer because the code
is read differently by each browser. The validation was a little bit tricky at some times since I am so new to
the HTML language. Sometimes, an error would come up and I would not understand what was wrong, so I would have
to tinker with the code until the error went away. After validation, we conducted usability tests. I enjoyed
this because it was a good experience on how to improve my own website. I reviewed the Yale Art School website,
and it was very hard to use. Now I know how I can make my website more user-friendly based on what I was able to
do with this website.
